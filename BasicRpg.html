Basic RPG

I tried to make a generic rpg in a low poly style.
Using Blender for Modeling / Texture
Unity as a Game Engine and VS2019 for Code.

Work time : ~40h (unfinished project)
Models
Animation
Coding
Implementation
Bug Fixing

Normaly this was my first project for the university portofolio, I thought that will be a great way to show the basic skills in the area of game development, like 3d Modeling and Coding. Previously I made a bunch of basic games, mostly text-based, to test my abilites of story writing and game design and also I did some 2D ones, copies of retro games like Space Invaders and Mario using GameMaker engine, unfortunately I don't have any of the games on this hard drive.
This game started in day 1 and finished in day ##, beacuse I've realised that I can not finished the game on the time that I've proposed to myself, beacause it had countless bugs.


The skills that I use making this game were:

  • Basic 3D Modeling. (low poly)
  • Texture knowledge. (UV Unwraping / Using Color Map)
  • Coding (C#)
  • Basic Unity Knowledge
  • 3D Animation

There is room for improvment? Actualy , there is alot of room for improvement. After this failed attempt I've found out that making games, isn't an easy task, but is quite a challenging and rewarding one. The ammount of happines I've got from seeing the animation and models combined in a game engine and responsive to key imputs was incompresable. I'm looking foward in learning Unity, beacuse it will help me making my own games and also help me visualise Indoor Arhitecture.

Did I learn something from making this model? More than in any project.

  1. Making Animation in Blender.
  2. Using Unity
  3. Making Models / Texture Optimized for Low Spec PC / Mobile Phones
  4. Working with Color Maps.
  5. Making Ui Elements
  6. Unity A.I Navigation
  7. Making Low Poly Characters / Understanding Low Poly Style
  8. Making Low Poly Weapons / Armors
